# Common Mistakes

1. Not reading the full requirements before starting.
2. Forgetting to run tests after changes.
3. Committing sensitive data (keys, passwords).
4. Not following the project's coding style.
5. Over-engineering simple solutions.
6. Ignoring error handling.
7. Not documenting complex logic.
8. Hardcoding configuration values.
9. Not using version control properly.
10. Skipping code reviews.
